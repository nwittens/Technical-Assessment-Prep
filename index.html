Background

Betterment customers hold investment portfolios containing multiple positions across different securities. At the end of each trading day, we need to calculate each customer's realized and unrealized gain/loss across their portfolio and persist a daily snapshot to the database.

You are building the DailyGainLossService — a Spring Boot service that reads position and price data from Postgres, computes gain/loss figures, and writes a summary report back to the database. This is production-level code that must handle real-world edge cases.

The data volume is ~50,000 customers with an average of 12 positions each. Performance matters.

CREATE TABLE positions (
  id              BIGSERIAL PRIMARY KEY,
  customer_id     BIGINT NOT NULL,
  security_id     VARCHAR(12) NOT NULL,   -- e.g. 'AAPL', 'VTI'
  quantity        NUMERIC(18,6) NOT NULL, -- fractional shares allowed
  account_type    VARCHAR(20) NOT NULL,   -- 'TAXABLE','IRA','ROTH'
  created_at      TIMESTAMPTZ DEFAULT NOW(),
  updated_at      TIMESTAMPTZ DEFAULT NOW(),
  CONSTRAINT uq_customer_security
    UNIQUE (customer_id, security_id, account_type)
);
CREATE TABLE daily_prices (
  id              BIGSERIAL PRIMARY KEY,
  security_id     VARCHAR(12) NOT NULL,
  price_date      DATE NOT NULL,
  closing_price   NUMERIC(18,4) NOT NULL,
  prev_close      NUMERIC(18,4),           -- nullable on first day
  CONSTRAINT uq_security_date
    UNIQUE (security_id, price_date)
);
CREATE INDEX idx_prices_date ON daily_prices (price_date, security_id);
CREATE TABLE tax_lots (
  id              BIGSERIAL PRIMARY KEY,
  customer_id     BIGINT NOT NULL,
  security_id     VARCHAR(12) NOT NULL,
  quantity        NUMERIC(18,6) NOT NULL,
  cost_basis      NUMERIC(18,4) NOT NULL, -- price per share at purchase
  acquired_date   DATE NOT NULL,
  account_type    VARCHAR(20) NOT NULL,
  closed          BOOLEAN DEFAULT FALSE   -- true if position sold
);
CREATE INDEX idx_lots_customer_security
  ON tax_lots (customer_id, security_id) WHERE closed = FALSE;
CREATE TABLE gain_loss_report (
  id               BIGSERIAL PRIMARY KEY,
  customer_id      BIGINT NOT NULL,
  report_date      DATE NOT NULL,
  security_id      VARCHAR(12) NOT NULL,
  account_type     VARCHAR(20) NOT NULL,
  quantity         NUMERIC(18,6) NOT NULL,
  market_value     NUMERIC(18,4) NOT NULL, -- quantity × closing_price
  cost_basis_total NUMERIC(18,4) NOT NULL, -- sum of lot cost_basis × qty
  unrealized_gl    NUMERIC(18,4) NOT NULL, -- market_value - cost_basis_total
  day_change       NUMERIC(18,4),           -- nullable if no prev_close
  is_long_term     BOOLEAN NOT NULL,        -- held > 1 year (tax status)
  generated_at     TIMESTAMPTZ DEFAULT NOW(),
  CONSTRAINT uq_report_row
    UNIQUE (customer_id, report_date, security_id, account_type)
);
Requirements
R1
Load positions and today's prices DB
Fetch all active positions joined with today's closing price from daily_prices. Skip any security with no price for today — log it, do not fail the job.
R2
Calculate unrealized gain/loss ALGO
For each position, sum the weighted cost basis across all open tax lots. Unrealized G/L = (quantity × closing price) − total cost basis. Use FIFO lot ordering.
R3
Calculate day change ALGO
Day change = quantity × (closing_price − prev_close). If prev_close is null, set day_change to null — do not default to zero.
R4
Determine long-term tax status ALGO
A position is long-term if the earliest open lot's acquired_date is more than 365 days before today. Use the lot table — do not hardcode or approximate.
R5
Batch upsert to gain_loss_report DB PERF
Write results using a single batch upsert (INSERT ... ON CONFLICT DO UPDATE). Do not loop individual inserts. Re-running for the same date must be idempotent.
R6
Handle scale gracefully PERF
50,000 customers × 12 positions = 600,000 rows. Process in chunks — do not load the entire dataset into memory at once.
